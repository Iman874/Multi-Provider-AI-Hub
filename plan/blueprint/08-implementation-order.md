# 08 — Implementation Order

---

## Build Sequence

Urutan implementasi dirancang agar setiap step bisa **ditest secara independen** sebelum lanjut ke step berikutnya.

---

### Phase 1: Foundation (Day 1)

#### Step 1.1 — Project Scaffolding
- [ ] Init project: `pyproject.toml`, `requirements.txt`
- [ ] Create folder structure (`app/`, semua sub-packages)
- [ ] Create all `__init__.py` files
- [ ] Setup `.env.example` dan `.gitignore`

#### Step 1.2 — Configuration
- [ ] Implement `app/config.py` (Settings class)
- [ ] Test: load `.env`, validate values

#### Step 1.3 — Core Infrastructure
- [ ] Implement `app/core/exceptions.py` (all custom exceptions)
- [ ] Implement `app/core/logging.py` (loguru setup)
- [ ] Implement `app/core/middleware.py` (request logging)

#### Step 1.4 — Schemas
- [ ] Implement `app/schemas/common.py`
- [ ] Implement `app/schemas/requests.py`
- [ ] Implement `app/schemas/responses.py`

**✅ Checkpoint**: Project runs, config loads, schemas validate.

---

### Phase 2: Provider Layer (Day 2)

#### Step 2.1 — Base Provider
- [ ] Implement `app/providers/base.py` (ABC)

#### Step 2.2 — Ollama Provider
- [ ] Implement `app/providers/ollama.py`
- [ ] `generate()` — text only first
- [ ] `stream()` — NDJSON parsing
- [ ] `embedding()` — `/api/embed`
- [ ] `supports_image()` 
- [ ] Test: generate text with local Ollama

#### Step 2.3 — Gemini Provider
- [ ] Implement `app/providers/gemini.py`
- [ ] `generate()` — text + multimodal
- [ ] `stream()` — streaming via SDK
- [ ] `embedding()` — embed_content
- [ ] `supports_image()`
- [ ] Test: generate text with Gemini API key

#### Step 2.4 — Provider Factory
- [ ] Implement `create_provider()` in `app/providers/__init__.py`

**✅ Checkpoint**: Both providers work independently via unit tests.

---

### Phase 3: Service Layer (Day 3)

#### Step 3.1 — Model Registry
- [ ] Implement `app/services/model_registry.py`
- [ ] `register()`, `get_model()`, `list_models()`
- [ ] `register_defaults()` with known models
- [ ] Test: register and lookup models

#### Step 3.2 — Generator Service
- [ ] Implement `app/services/generator.py`
- [ ] `generate()` — route + validate + normalize
- [ ] `stream()` — route + yield tokens
- [ ] `embedding()` — route + validate capability
- [ ] Test: full flow with mocked providers

**✅ Checkpoint**: Service layer passes all unit tests.

---

### Phase 4: API Layer (Day 4)

#### Step 4.1 — Dependencies
- [ ] Implement `app/api/dependencies.py`
- [ ] `initialize_services()` function

#### Step 4.2 — Endpoints
- [ ] `POST /api/v1/generate` — `app/api/endpoints/generate.py`
- [ ] `POST /api/v1/stream` — `app/api/endpoints/stream.py`
- [ ] `GET /api/v1/models` — `app/api/endpoints/models.py`
- [ ] `POST /api/v1/embedding` — `app/api/endpoints/embedding.py`

#### Step 4.3 — Router & Main
- [ ] Implement `app/api/router.py`
- [ ] Implement `app/main.py` (startup, shutdown, exception handlers)

**✅ Checkpoint**: All endpoints respond correctly via Swagger UI.

---

### Phase 5: Integration & Polish (Day 5)

#### Step 5.1 — Image Utilities
- [ ] Implement `app/utils/image.py`
- [ ] Base64 encoding/decoding
- [ ] URL → base64 conversion
- [ ] MIME type detection

#### Step 5.2 — Multimodal Testing
- [ ] Test image input with Ollama vision model
- [ ] Test image input with Gemini
- [ ] Verify abstraction works seamlessly

#### Step 5.3 — Streaming Testing
- [ ] Test SSE streaming with Ollama
- [ ] Test SSE streaming with Gemini
- [ ] Test with curl / Postman / browser

#### Step 5.4 — Error Scenarios
- [ ] Test provider not found
- [ ] Test model not found
- [ ] Test capability mismatch
- [ ] Test provider timeout
- [ ] Test provider connection failure

#### Step 5.5 — Documentation
- [ ] README.md with setup instructions
- [ ] API docs auto-generated by FastAPI (Swagger)
- [ ] Example curl commands

**✅ Final**: Fully functional, tested, documented API.

## Quick Validation Commands

```bash
# Start server
uvicorn app.main:app --reload --port 8000

# Test generate
curl -X POST http://localhost:8000/api/v1/generate \
  -H "Content-Type: application/json" \
  -d '{"provider":"ollama","model":"llama3.2","input":"Hello world"}'

# Test models
curl http://localhost:8000/api/v1/models

# Test streaming
curl -X POST http://localhost:8000/api/v1/stream \
  -H "Content-Type: application/json" \
  -d '{"provider":"ollama","model":"llama3.2","input":"Tell me a joke"}'

# Test embedding
curl -X POST http://localhost:8000/api/v1/embedding \
  -H "Content-Type: application/json" \
  -d '{"provider":"ollama","model":"nomic-embed-text","input":"Hello world"}'

# Swagger UI
# Open: http://localhost:8000/docs
```
