## 1. Judul Task

Implementasi `app/schemas/responses.py` — Pydantic models untuk semua response bodies (GenerateResponse, ModelInfo, EmbeddingResponse, UsageInfo, ErrorResponse).

---

## 2. Deskripsi

Membuat Pydantic BaseModel untuk setiap jenis response yang akan dikirim oleh API. Schemas ini menjamin **normalisasi output** — apapun provider-nya, response selalu punya format yang sama. Juga digunakan oleh Swagger UI untuk auto-dokumentasi.

---

## 3. Tujuan Teknis

- `UsageInfo` — token usage (prompt, completion, total)
- `GenerateResponse` — standard response untuk text generation
- `ModelInfo` — metadata model untuk GET /models
- `EmbeddingResponse` — vector embedding result
- `ErrorResponse` — standard error format
- Semua response memiliki `response_model` yang jelas di endpoint

---

## 5. Langkah Implementasi

### Step 1: Buat `app/schemas/responses.py`

```python
"""
Response schemas for AI Generative Core API.

These Pydantic models define the structure of all API responses.
All providers MUST normalize their output to match these schemas,
ensuring consistent responses regardless of the underlying provider.
"""

from typing import Any, Optional

from pydantic import BaseModel, Field


class UsageInfo(BaseModel):
    """
    Token usage information returned by AI providers.

    Not all providers return all fields — hence all are optional.
    """

    prompt_tokens: Optional[int] = Field(
        default=None, description="Number of tokens in the prompt"
    )
    completion_tokens: Optional[int] = Field(
        default=None, description="Number of tokens in the completion"
    )
    total_tokens: Optional[int] = Field(
        default=None, description="Total tokens used (prompt + completion)"
    )


class GenerateResponse(BaseModel):
    """
    Standard response for text/multimodal generation.

    All providers normalize their output to this format.
    """

    output: str = Field(
        ..., description="Generated text output"
    )
    provider: str = Field(
        ..., description="Provider that generated the response"
    )
    model: str = Field(
        ..., description="Model that generated the response"
    )
    usage: Optional[UsageInfo] = Field(
        default=None, description="Token usage statistics"
    )
    metadata: Optional[dict[str, Any]] = Field(
        default=None, description="Additional provider-specific metadata"
    )


class ModelInfo(BaseModel):
    """
    Model metadata returned by GET /models endpoint.

    Describes a model's identity and capabilities.
    """

    name: str = Field(
        ..., description="Model name", examples=["llama3.2", "gemini-2.0-flash"]
    )
    provider: str = Field(
        ..., description="Provider name", examples=["ollama", "gemini"]
    )
    supports_text: bool = Field(
        ..., description="Whether the model supports text generation"
    )
    supports_image: bool = Field(
        ..., description="Whether the model supports image/multimodal input"
    )
    supports_embedding: bool = Field(
        ..., description="Whether the model supports vector embedding"
    )


class EmbeddingResponse(BaseModel):
    """
    Response for vector embedding generation.
    """

    embedding: list[float] = Field(
        ..., description="Embedding vector as a list of floats"
    )
    provider: str = Field(
        ..., description="Provider that generated the embedding"
    )
    model: str = Field(
        ..., description="Model that generated the embedding"
    )


class ErrorResponse(BaseModel):
    """
    Standard error response format.

    Used for documenting error responses in Swagger UI.
    Actual error responses are generated by the exception handler in main.py.
    """

    error: str = Field(
        ..., description="Human-readable error message"
    )
    code: str = Field(
        ..., description="Machine-readable error code",
        examples=["PROVIDER_NOT_FOUND", "MODEL_NOT_FOUND"],
    )
    detail: Optional[str] = Field(
        default=None, description="Additional error details"
    )
```

### Step 2: Verifikasi

```bash
python -c "
from app.schemas.responses import (
    UsageInfo, GenerateResponse, ModelInfo, EmbeddingResponse, ErrorResponse
)

# GenerateResponse
resp = GenerateResponse(
    output='Hello! How can I help?',
    provider='ollama',
    model='llama3.2',
    usage=UsageInfo(prompt_tokens=5, completion_tokens=10, total_tokens=15),
    metadata={'duration_ms': 1234},
)
print(resp.model_dump_json(indent=2))

# ModelInfo
model = ModelInfo(
    name='llama3.2',
    provider='ollama',
    supports_text=True,
    supports_image=False,
    supports_embedding=False,
)
print(model.model_dump())

# EmbeddingResponse
emb = EmbeddingResponse(
    embedding=[0.1, 0.2, 0.3],
    provider='ollama',
    model='nomic-embed-text',
)
print(f'embedding length: {len(emb.embedding)}')
"
```
